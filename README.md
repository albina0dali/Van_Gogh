# 🌾 SubsiSmart KZ

**AI-система скоринга для государственных субсидий Казахстана**

Разработано для **Decentrathon 5.0** | Трек: **AI for Government**

---

## 📋 Описание проекта

SubsiSmart KZ — это интеллектуальная система анализа и скоринга заявок на государственные субсидии для сельхозпроизводителей Казахстана. Система использует методы машинного обучения для:

- 🎯 **Сегментации заявителей** с помощью алгоритма K-Means
- 🚨 **Выявления аномалий** и заявок с высоким риском
- 📊 **Регионального анализа** эффективности распределения субсидий
- 🔍 **Прозрачного скоринга** с объяснением факторов риска

---

## 🎯 Решаемая проблема

Сегодня государственные субсидии распределяются по принципу «первый подал — первый получил». Это приводит к:
- Неэффективному использованию бюджетных средств
- Отсутствию учета реальной эффективности хозяйств
- Сложности в выявлении мошеннических схем

**SubsiSmart KZ** переводит процесс на data-driven подход с прозрачными критериями оценки.

## 🚀 Функционал

### 1. **Предобработка данных**
- Преобразование даты в признаки (месяц, день недели, час, квартал)
- Кодирование категориальных признаков (область, направление деятельности)
- Автоматическая очистка и нормализация данных

### 2. **K-Means кластеризация**
- Сегментация заявителей на 4 группы по признакам:
  - Норматив субсидии
  - Причитающая сумма
- Анализ характеристик каждого кластера

### 3. **Детектирование аномалий**
- Статистический анализ превышений (>3 стандартных отклонений)
- Автоматическая маркировка заявок как "High Risk"
- Проверка соотношения причитающей суммы к нормативу

### 4. **Региональный анализ**
- Рейтинг регионов по проценту исполненных заявок
- Анализ эффективности распределения средств
- Выявление лучших практик

### 5. **Risk Scoring**
- Многофакторная оценка риска:
  - Аномальные суммы
  - Статус заявки
  - Время подачи
  - Величина субсидии
- Шкала от 0 до 100 баллов

### 6. **Веб-интерфейс**
- Интерактивные дашборды с графиками
- Фильтрация и поиск заявок
- Визуализация рисков и трендов
- Минималистичный дизайн (бело-зеленая гамма)

## 📦 Установка

### Требования
- Python 3.8+
- pip

### Шаги установки

1. **Клонируйте репозиторий**
```bash
cd SubsiSmartKZ
```

2. **Создайте виртуальное окружение (рекомендуется)**
```bash
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\Scripts\activate  # Windows
```

3. **Установите зависимости**
```bash
pip install -r requirements.txt
```

4. **Подготовьте данные**
- Поместите файл `Выгрузка_по_выданным_субсидиям_2025_год__обезлич_.xlsx` в папку `data/`

---

## 🎮 Запуск

### Вариант 1: Консольный анализ

```bash
python scoring_engine.py
```

Выполняет полный анализ данных и выводит результаты в консоль:
- Статистика обработки
- Характеристики кластеров
- Распределение рисков
- Региональный отчет

Результаты сохраняются в папку `results/`:
- `processed_data.csv` — обработанные данные со всеми признаками
- `regional_report.csv` — региональная статистика

### Вариант 2: Веб-приложение

```bash
python app.py
```

Откройте браузер и перейдите по адресу: **http://localhost:5000**

## 🎓 Алгоритмы

### K-Means Кластеризация
- **Количество кластеров**: 4
- **Признаки**: Норматив, Причитающая сумма
- **Предобработка**: StandardScaler для нормализации
- **Цель**: Сегментация заявителей по объему запрашиваемых средств

### Детектирование аномалий
- **Метод**: Статистический анализ (3-sigma правило)
- **Группировка**: По направлениям деятельности
- **Критерии**:
  - Превышение > среднее + 3×σ
  - Соотношение сумма/норматив > 3
  - Нулевая сумма при ненулевом нормативе

### Risk Scoring (взвешенная сумма)
```
Risk Score = 
  + 50 (High Risk аномалия)
  + 25 (Medium Risk аномалия)
  + 30 (Отклоненная заявка)
  + 20 (Отозванная заявка)
  + 10 (Ночная подача)
  + 15 (Топ-5% по сумме)
```

## 🔒 Explainability (Объяснимость)

Система обеспечивает полную прозрачность решений:

1. **Для каждого кластера** показаны средние значения норматива и суммы
2. **Для каждой аномалии** указан тип превышения и величина отклонения
3. **Risk Score** разложен на компоненты (видно, какие факторы повлияли)
4. **Региональная статистика** показывает процент успеха по каждому региону

**Важно**: Система помогает принимать решения, но финальное слово остается за экспертом.

## ⚠️ Ограничения

1. **Качество данных**: Система зависит от полноты и корректности входных данных
2. **Исторические данные**: Анализ основан на данных 2025 года, тренды могут измениться
3. **Обобщения**: K-Means предполагает, что кластеры имеют сферическую форму
4. **Пороги риска**: 3-sigma правило может требовать настройки для разных направлений
5. **Не заменяет эксперта**: Система — инструмент поддержки, не автоматизация решений
